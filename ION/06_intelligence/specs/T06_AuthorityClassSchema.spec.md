# AuthorityClassSchema — Artifact Trust Classification

## 1. PURPOSE

Every artifact in ION must be classified for how much trust agents and the daemon
should place in it. This prevents the failure mode that killed prior consolidation
attempts: generated state being mistaken for authority, witness material being
treated as current law, and plans being treated as proof of runtime reality.

From GPT 5.4 blueprint §2.6 and §14:
> "The system must distinguish among Authority, Witness, Plan, Audit, Generated State, and Stale Competitor."

## 2. SCHEMA DEFINITION

```yaml
AuthorityClassSchema:
  schema_version: "1.0"

  AuthorityClass:
    # --- The seven classes, ordered from most to least authoritative ---
    
    AUTHORITY:
      description: "Currently governing truth. May influence daemon sequencing and future context directly."
      examples:
        - "Ratified constitution (01_doctrine/SOVEREIGN_CONSTITUTION.md with status: ACTIVE)"
        - "Protected agent registry (03_registry/agent_registry.json)"
        - "Live runtime code that is actually executing (heartbeat.py in the running daemon)"
      trust_level: "Agents and daemon treat this as ground truth."
      mutation_rule: "Only Sovereign or governed write pipeline may modify."
      
    WITNESS:
      description: "Useful evidence of prior state or lineage. Must not be treated as current authority."
      examples:
        - "ION-BUILD/src/ion/kernel/model.py (older kernel, not the unified ION version)"
        - "ProjectOpus/21_ARCHAEOLOGY_REMAP/* (historical analysis, not current law)"
        - "Capsule-based roster claims that have drifted from registry truth"
      trust_level: "Agents may cite as evidence but must not use as governing input."
      mutation_rule: "May be moved to archive. Must not be silently deleted."
      
    PLAN:
      description: "Intended direction, not proof of runtime reality."
      examples:
        - "ION/PLAN.md (the consolidation plan — intent, not proof)"
        - "SOS/02_architecture/SOS_CONSOLIDATION_PLAN.md (historical plan)"
        - "Any task file describing work to be done"
      trust_level: "Agents follow plans but do not treat them as evidence that the work is done."
      mutation_rule: "Plans evolve through revision cycles. Prior versions are witness material."
      
    AUDIT:
      description: "Interpretation of state at a point in time. Must be revalidated."
      examples:
        - "ION/06_intelligence/audits/* (Nemesis audit findings)"
        - "00_CONSOLIDATED_ATLAS/* (the GPT 5.4 forensic audit)"
        - "Any compliance check, gap analysis, or finding document"
      trust_level: "Agents may cite findings but must verify claims against current state."
      mutation_rule: "Audits are immutable once filed. New audits supersede old ones."
      
    GENERATED_STATE:
      description: "Auto-produced system summaries, route snapshots, compiled context. Useful but derivative."
      examples:
        - "CAPSULE.md work log entries (generated by agents, not canonical roster)"
        - "HOT_CONTEXT.md (compiled by distiller, not manually authored)"
        - "system_ledger.json (daemon receipts, not governing truth)"
        - "STATUS.md (agent self-reports, not verified state)"
      trust_level: "Useful for operational awareness. Must not be cited as authority."
      mutation_rule: "May be regenerated at any time. Prior versions need not be preserved."
      
    STALE_COMPETITOR:
      description: "Artifact that resembles authority strongly enough to mislead agents."
      examples:
        - "Project-Gemini/ion-core/* (near-identical Victus fork with broken paths)"
        - "SOS/04_packages/heartbeat/src/daemon.ts (TS daemon vs live Python heartbeat)"
        - "ION-BUILD/.ion-context/MANIFEST.md vs ION-BUILD/context/ION_MANIFEST.md (duplicated manifest)"
      trust_level: "DANGEROUS. Agents must be explicitly warned. May cause authority confusion."
      mutation_rule: "Should be archived with explicit stale marker. Must never be silently promoted."
      
    ARCHIVE_REFERENCE:
      description: "Historical material preserved for lineage. Not evidence of present state."
      examples:
        - "SOS/00_archive/* (historical build audits)"
        - "ION/00_archive/* (preserved witness material from consolidation)"
        - "Any file explicitly moved to an archive directory"
      trust_level: "Read-only reference. Never treated as current."
      mutation_rule: "Immutable. Archive is append-only."

  # --- Artifact annotation ---
  ArtifactClassification:
    artifact_path: string        # REQUIRED
    authority_class: AuthorityClass  # REQUIRED
    classified_by: string        # REQUIRED — agent or process that made the classification
    classified_at: string        # REQUIRED — ISO 8601
    justification: string        # REQUIRED — why this class was assigned
    last_revalidated: string     # OPTIONAL — when was this classification last confirmed
    competition_refs: list[string]  # OPTIONAL — 05A competition rows if class is contested
```

## 3. CLASSIFICATION RULES

1. **Default class is GENERATED_STATE** unless explicitly promoted
2. **Only Sovereign may promote to AUTHORITY** (ratification)
3. **Audits may recommend reclassification** but cannot execute it
4. **STALE_COMPETITOR must be actively identified** — it doesn't auto-classify
5. **WITNESS and ARCHIVE_REFERENCE are permanent** — they do not decay further
6. **PLAN artifacts that are executed do not become AUTHORITY** — the executed output does

## 4. VALIDATION CRITERIA

- [ ] Every artifact in the unified ION root should be classifiable
- [ ] Classification is persistent (stored alongside or indexed)
- [ ] STALE_COMPETITOR identification prevents the contagion problem from ProjectOpus archaeology
- [ ] GENERATED_STATE vs AUTHORITY distinction prevents capsule/ledger confusion from 05A findings
- [ ] Classification influences ContextPackage compilation (AUTHORITY weighted higher than WITNESS)
