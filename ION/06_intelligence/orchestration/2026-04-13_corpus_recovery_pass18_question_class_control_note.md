# Pass 18 — Canonical future-work question classes and defaults

## What this pass adds

This pass adds the first small canonical set of future-work question classes for the atlas operating layer.

New surfaces:
- `corpus_recovery/00_program/canonical_future_work_question_classes.md`
- `corpus_recovery/00_program/canonical_future_work_question_class_defaults.csv`

## Why this matters

The atlas already had:
- a working center-selection guide
- a future-work gating authority matrix
- packet requirements
- posture selection guidance
- anti-reinvention law
- worked start examples

What it still lacked was one stable answer to:
- what kind of question is this, first?

Without that, future work could still begin from a plausible but distorted framing.

This pass reduces that problem by defining a small set of canonical question classes and attaching each one to:
- a default posture
- a primary preserved center
- a widening default

## Current interpretation

The atlas operating layer should now be read in this order when starting future work:
1. classify the question through the canonical question classes
2. consult the question-class defaults matrix
3. follow the atlas control surface and work gate
4. only then begin a lawful work-start record

## Status

This is a control-layer stabilization pass.
It does not claim final ontology.
It provides a stronger default discipline for future work.
