# Template Surface Change: System Lineage Profile

## Purpose

Create one current-phase report template for profiling a root, build family, or
preservation line during controlled reintegration work.

The current template floor already supports general research, evidence, and
proposal work. What it does not yet provide is a compact, repeatable profile
shape for answering the recurring foundry question:

what does this root preserve best, what does it fail to preserve, how should it
be classified, and what are the main misread risks?

## Historical Search / Provenance

The recovery program already contains many lineage-profile-like documents under
`corpus_recovery/03_system_profiles/`, but there is no explicit shared template
for producing new ones consistently.

This template is therefore:

- not a reinvention of a missing historical constitutional packet family,
- not a widening of the canonical packet floor,
- and not an attempt to ratify final canon.

It is a current-phase support template extracted from repeated recovery-program
practice.
