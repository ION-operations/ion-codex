# Evented Template File Graph Recovery Packet

## 1. Recovery Finding

The prior restoration packet recovered ION as a living context graph. This packet restores the missing mechanical activation law: the graph is living because template-instantiated files can become validated event surfaces.

A completed file is not only documentation. If it is created from a known template and passes its completion threshold, it can be validated, indexed, routed, scheduled, warned against, settled, or used to activate specialists. That evented behavior is what turns a file graph into an operating substrate.

## 2. Restored Doctrine

ION is a living context graph materialized as an evented, template-instantiated file system.

Files are durable graph objects. Templates define lawful shape and automation behavior. Agents and humans fill templates using bounded context packages. When completed files pass validation, daemons and schedulers trigger lawful graph reactions. Receipts prove every mutation, warning, scheduling action, settlement, or refusal.

Without this law, ION risks becoming a static graph of context objects. With this law, ION becomes reactive:

- completed dependency-change packets warn dependent systems;
- completed summaries refresh context-package sources;
- completed role handoffs update continuity lanes;

This is the operational bridge between graph substrate and daemon/scheduler/indexer behavior.
