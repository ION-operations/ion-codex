# First-Contact, Project Identity, and Collaboration Model v2.6.7

## First Contact

At first meaningful turn, establish one of:

- existing project context package,
- new project package draft,
- explicit no-project exploratory lane.

## Project Identity

Use a stable project ID/tag per collaboration thread. Candidate examples:

- `daimon:<project_slug>:<date_or_epoch>`
- `ion_project:<team_or_user>:<slug>`

## Collaboration Scope Layers

- Base scope: project-bound continuity.
- Capability scope: what each participant/agent may do.
- Invite scope: who may join, view, propose, or settle state.

## Settlement Rule

Generated output remains candidate until classified, receipted, and linked to
project continuity artifacts.
