# dAimon Project Doctrine v2.6.7

## Core Position

ION/dAimon names the failure clearly: AI output is being mistaken for state.

dAimon posture:

- AI creates candidate work.
- dAimon governs what candidate work is allowed to become.
- Candidate -> classified -> settled -> receipted -> auditable continuation.

## Product Thesis

dAimon is productized ION targeting the Layer 3 trust layer through a
contest-bounded vertical slice.

dAimon removes re-explanation tax by preserving project continuity outside any
single chat instance.

## Proven Relay Path

ChatGPT <-> ION MCP <-> Codex Capsule <-> Codex work queue <-> proof return is
now a proven routine relay path. Manual paste is optional, not required, for
routine coordination.

## First-Contact and Project Identity Model

- First contact should initialize or attach a project-bound package.
- Every project should carry a stable project ID/tag.
- Collaboration scope layers on top of project identity using capability/invite
  boundaries.
- Work should remain project-scoped; cross-project claims require explicit
  handoff and receipts.
