# What Is ION?

ION is a governed continuity layer for AI-assisted work.

Ordinary AI chat often makes the user become the operating system: the user must
keep reminding the assistant what matters, what is accepted, what is only a
guess, what needs proof, and what should happen next.

ION moves that burden into a structured operating layer:

```text
raw user intent
-> context
-> domain/work-area routing
-> workflow/template
-> candidate output
-> proof/check
-> acceptance when required
-> receipt/export
-> next state
```

The key law:

```text
AI output is not state.
```

An AI answer becomes durable only when it is grounded, proof-marked, accepted
where needed, receipted/exported, and carried forward.

ION uses:

- domains: work areas;
- agents: domain-local interfaces;
- templates/skills: governed workflows;
- packets: next work steps;
- receipts: proof notes / saved decisions;
- context packages: working memory;
- Persona: user-facing response boundary;
- Relay/Steward: semantic and acceptance boundaries.

The user should not have to speak ION jargon. The user should feel better
continuity, clearer next steps, and less prompt-management burden.
