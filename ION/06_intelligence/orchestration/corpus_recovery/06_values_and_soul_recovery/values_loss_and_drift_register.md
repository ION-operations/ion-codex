# Values loss and drift register

## V-001 — Template primacy weakened
- Old center: Aether/template-development line
- Current symptom: current branch preserves only an admitted minimum active template floor, not the fully restored canon
- Consequence: repeated false novelty and local template replacement

## V-002 — Truename over chassis repeatedly violated
- Old center: semantic/true-name law
- Current symptom: Codex repeatedly gained too much authority as if it were orchestration truth rather than common carrier alias
- Consequence: role/chassis confusion and startup drift

## V-003 — Law/history/memory separation weakened
- Old center: Aether constitutional center and atlas discipline
- Current symptom: historical startup surfaces repeatedly bled into active teaching truth
- Consequence: fresh operators could not tell live law from lineage witness

## V-004 — Activation honesty weakened
- Old center: capability honesty
- Current symptom: current branch often spoke as if it already had an integrated activation kernel, while activation remained partly manual and distributed across docs/handoffs
- Consequence: workflow confusion and orchestration overstatement

## V-005 — Atlas compression lost
- Old center: Aether atlas/kernel compression
- Current symptom: too many protocol/orchestration surfaces competing for center
- Consequence: project became globally illegible even when locally cleaner

## Additional first-pass drift note
A recurring pattern across the estate is that template-development and meta-template centers survived outside the current branch more strongly than the current branch itself admits. This is one of the clearest examples of how the project kept rebuilding partial local template order while losing sight of older stronger template-making centers.
