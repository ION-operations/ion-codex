# Worked examples layer

This layer holds concrete enacted example packets for review candidates that are already:
- delta-defined,
- design-landed,
- draft-reviewed,
- promotion-candidate reviewed,
- and counterexample stressed.

Its purpose is not to add new law.
Its purpose is to demonstrate that a review candidate survives real bounded flows without collapsing its boundaries.

Current focus:
- activation authority / executor lifecycle carrier-crossing examples
- activation authority / executor lifecycle receipt-settlement examples
- runtime/session receipt-linkage examples

This layer remains:
- non-ratified,
- review-support only,
- and insufficient by itself for thaw or active-law installation.
