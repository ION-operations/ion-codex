# Install-path mapping layer

This layer exists to answer one bounded question:

**If a promotion candidate leaves review space, where would it land in active architecture, and what existing surfaces would it border, refine, or leave untouched?**

This layer does not install new law.
It only maps:

- likely active-architecture landing paths
- adjacent existing protocols
- non-replacement boundaries
- staged promotion order
- thaw blockers that still remain after mapping

Current primary surfaces:
- `activation_lifecycle_install_path_mapping_packet.md`
- `activation_lifecycle_install_path_mapping_matrix.csv`
- `activation_lifecycle_install_path_mapping.md`
- `runtime_session_install_path_mapping_packet.md`
- `runtime_session_install_path_mapping_matrix.csv`
- `runtime_session_install_path_mapping.md`
