# Project Control Kernel Governance Card

## Use this card when the frozen kernel itself is in question

1. Check whether the issue is repeated or merely local.
2. Run the kernel health checks.
3. Choose the review layer:
   - event-driven
   - rolling health
   - milestone review
4. Record the review using the review record template.
5. If revision is warranted, enter the thaw/revision protocol.
6. If revision is not warranted, keep the kernel frozen and record why.

## Default rule

The kernel should remain frozen unless patterned insufficiency is shown.
Elegance, novelty, impatience, or convenience are not sufficient reasons to thaw it.
