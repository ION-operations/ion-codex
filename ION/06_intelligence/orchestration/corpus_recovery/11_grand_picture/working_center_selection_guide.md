# Working center selection guide

## Purpose

This guide tells future work **which preserved center to consult first** depending on the kind of question being asked.

## Selection rules

### If the question is about current branch truth
Start with:
- current extracted ION branch
- current atlas base

### If the question is about startup/routing/default packet law
Start with:
- current extracted ION branch

### If the question is about runtime/API/session execution
Start with:
- ION-BUILD
Then compare against:
- AETHER-OS-V4

### If the question is about activation, manager logic, orchestration, swarm, or agent-cycle execution
Start with:
- operation-victus
Then compare against:
- Project-Gemini
Then compare current branch activation-law claims against those historical executable centers

### If the question is about meta-template law or template-development constitutional strength
Start with:
- ION-BUILD historical TEMPLATE_DEVELOPMENT center
Then compare against:
- AETHER-OS-V4
- SOS family exact-match witnesses
- current branch template bridge repairs

### If the question is about compact executable memory
Start with:
- production ION pair
- IONv2
Then compare against:
- current extracted branch

### If the question is about prior atlas/consolidation attempts
Start with:
- 00_CONSOLIDATED_ATLAS
- ProjectOpus
- ATLAS
Then use wrapper roots only as supporting provenance witnesses

## Hard rule

Do not answer a cross-capability question from only one center when the atlas already says the truth is distributed across multiple preserved lines.

## Frequent mistakes to avoid

- using the current branch to answer historical runtime-center questions
- using wrapper roots as if they were primary centers
- treating Project-Gemini as wholly independent from Victus
- treating compact witnesses as obsolete merely because they are smaller
- treating bridge-era repairs as equal to historical constitutional centers without explicit comparison
