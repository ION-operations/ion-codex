# Default project control kernel

## Purpose

This file names the **default long-lived project control kernel**.

Use this when future work needs the smallest durable control start for the project as it now exists.

## Kernel expansion rule

Widen beyond this kernel only when one of the following is true:
- the defaults matrix points to a preserved center that must be read directly
- the conflict register shows a conflict that must be compared directly
- the question class or posture is still unclear
- the task needs the answer/output or landing/horizon control layers explicitly
- the task is recover-first and needs deeper evidence surfaces
- the task specifically needs a demonstration/example layer

## Why this is different from the earlier default operating kernel

The earlier default operating kernel proved there was a stable minimal control path.

This default **project** control kernel makes the stronger claim that the control path is now good enough to act as the default long-lived project work-start layer.

## Limits

## Freeze layer

The default project control kernel should now also be read with:
- `../00_program/project_control_kernel_state_card.md`
- `../00_program/project_control_kernel_freeze_judgment.md`
- `../00_program/project_control_kernel_freeze_manifest.csv`

This kernel does **not**:
- replace the wider atlas
- resolve preserved conflicts
- authorize final canon claims
- eliminate the need for center-specific recovery when the question demands it

It exists to make future work start lawfully, not to pretend the organism is fully reconciled.


## Escalation rule

After entering the default project control kernel, consult:
- `../00_program/project_control_kernel_escalation_judgment.md`
- `../00_program/project_control_kernel_escalation_matrix.csv`

The kernel should now explicitly decide whether the work may:
- stay inside the kernel
- widen into the broader atlas or preserved centers
- or stop and reassess before proceeding
