# Era 2 branch charter pack

## Purpose

This layer exists to make multi-chat / multi-branch acceleration lawful.

The project may now use multiple chat branches, but only under a constrained topology that preserves one canonical repo center.

## Core rule

Parallelize **review-space and evidence-space**. Keep **active-law emission and root command-surface control serial**.

That means branch chats may produce:

- delta packets
- surface-design packets
- quarantined review drafts
- counterexample reviews
- worked-example packets
- install-path maps
- promotion-candidate packets

Branch chats may **not** directly install active architecture or mutate root orchestration surfaces.

## Authorized topology

1. **Conductor chat**
   - only authority for orchestration board updates
   - only authority for root startup/orientation surfaces
   - only authority for active-law emission

2. **Lane C runtime/session branch**
   - current primary parallel work lane
   - review-entry and promotion-candidate work only

3. **Lane A meta-template branch**
   - secondary parallel lane
   - design/review work only while Lane C is selected

4. **Adversarial audit branch**
   - no new canon creation
   - pressure-tests outputs from Lane C and Lane A

## Required shared packet

Every branch must begin from the same command packet:

- current orchestration board
- last completed objective
- selected lane
- horizon
- success condition
- out-of-bounds set
- expected output type
- stop condition

## Output discipline

Every branch deliverable must name:

- objective
- allowed files
- forbidden files
- current horizon
- exact output class
- success condition
- redirect condition
- handoff recommendation

## Current judgment

This pack authorizes branching as a speed tactic, but does **not** authorize distributed canon.

One repo center remains authoritative.
