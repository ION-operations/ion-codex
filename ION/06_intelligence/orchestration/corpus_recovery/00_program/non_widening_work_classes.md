# Non-Widening Work Classes

## Purpose

This file names the work classes that should normally **not** widen the organism.

These are the classes where the default lawful postures are:
- consult only
- compare centers
- recover first

not bounded extension.

---

## Default non-widening classes

### A. Center identification
Questions like:
- which preserved center should answer this?
- where did this capability live best?

Default posture:
- consult only
- compare centers

### B. Historical lineage clarification
Questions like:
- is this root a predecessor, sibling, wrapper, continuation, or mirror?

Default posture:
- compare centers
- recover first

### C. Values and soul questions
Questions like:
- what did ION/Aether actually value?
- where did value precedence collapse?

Default posture:
- consult only
- compare centers
- recover first

### D. Meta-template and constitutional center questions
Questions like:
- where did template development actually live?
- how does current repair differ from historical meta-template strength?

Default posture:
- compare centers
- recover first

### E. Wrapper / provenance interpretation
Questions like:
- is this a primary center or a wrapper witness?
- how should this audit package be read?

Default posture:
- compare centers
- recover first

### F. Compactness vs explicitness tensions
Questions like:
- is compactness stronger here or just more compressed?
- is the current branch broader because it is better or because it is heavier?

Default posture:
- compare centers

---

## When widening may become lawful

A non-widening class may become bridge repair or bounded extension only when:
- the atlas gate was completed
- the relevant centers were consulted
- conflict posture was made explicit
- the work-start record explains why consult/compare/recover is insufficient

Without that, widening should be treated as likely reinvention.
