# Project control kernel state card

## Purpose

This is the **smallest one-page state card** for the default project control kernel.

Use it when future work needs the fastest honest reminder of:
- what the kernel is for,
- what state it expects at work start,
- what decisions it controls,
- and when it must widen, return, or stop.

## Current kernel state

### Status
- default project control kernel: **active**
- stability: **stable enough for default work-start control**
- canon status: **non-final**
- conflict posture: **preserve conflicts; do not flatten them**

### Kernel purpose
The kernel exists to control how project work begins.
It is responsible for:
1. classifying the question
2. selecting the default preserved center
3. selecting the default posture
4. selecting the narrowest lawful answer/output class
5. selecting the lawful landing boundary
6. checking whether work should stay, widen, or stop
7. recording the start honestly

### Default control sequence
- question class
- primary center
- posture
- answer/output class
- landing boundary
- horizon / completion understanding
- stay / widen / stop
- if widened: return / reassess / record

### Widen only when
- the defaults matrix points to a preserved center that must be read directly
- preserved conflicts require direct comparison
- recover-first work genuinely needs deeper evidence
- the kernel cannot classify the task honestly without widening

### Stop and reassess when
- question class becomes unstable
- landing boundary becomes invalid
- conflict pressure deepens unexpectedly
- reinvention risk appears
- widened work cannot satisfy a lawful return condition

### What the kernel is not
It is not:
- final canon
- total organism closure
- a reason to ignore the wider atlas
- permission to erase preserved contradictions

## Practical reading

If a future worker only has time for one page before beginning, read this card first, then the defaults matrix, then the conflict register, then the packet-gate checklist.
