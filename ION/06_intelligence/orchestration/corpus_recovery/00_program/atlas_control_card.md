# Atlas control card

## Purpose

This is the smallest practical control card for future work.
Use it when you need to begin work without drifting, widening too early, or starting from the wrong preserved center.

## Six-step control path

1. **Classify the question** using `canonical_future_work_question_classes.md`.
2. **Read the default row** in `future_work_control_defaults_matrix.csv`.
3. **Consult the primary preserved center** named there before widening.
4. **Check preserved conflicts** in `12_status_and_conflicts/conflict_register.md` and `missing_evidence_register.md`.
5. **Choose the narrowest lawful output class** and landing boundary.
6. **Record the start** with `atlas_gated_work_start_record_template.md` or `atlas_orchestration_horizon_record_template.md`.

## Minimum fields future work should know before it proceeds

- question class
- primary center
- default posture
- default answer/output class
- default landing boundary
- widening status
- current process position
- near / mid / far horizon
- completion condition for this pass
- reassessment trigger

## Core warning

If the question is framed incorrectly, the wrong center is consulted, or the output class is too wide, the work is already drifting before any code or documents change.
