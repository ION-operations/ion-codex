# Bounded Extension vs Illegal Reinvention

## Purpose

This file defines the difference between lawful future extension and the project's main historical failure mode: false replacement.

---

## Lawful bounded extension

A work item counts as a lawful bounded extension only if all of the following are true:

1. The relevant preserved centers were consulted first.
2. The estate was searched for prior versions of the same burden.
3. The work either:
   - restores a lost preserved center,
   - reconciles two preserved centers,
   - bridges a known gap explicitly recorded in the atlas,
   - or adds a narrow surface that appears genuinely absent.
4. The output does not silently erase predecessor meaning.
5. The packet declares why this is not a reinvention.

Examples:
- recovering a lost activation receipt path from the historical manager line
- bridging current startup/routing law to a historically stronger activation center
- restoring a template-development surface that is explicitly missing from the current branch

---

## Illegal reinvention

A work item should be treated as illegal reinvention if one or more of these are true:

1. It authors a new local system without first checking the atlas.
2. It builds a new manager/orchestrator/template/protocol because the current branch feels incomplete, while older lines already preserve that burden.
3. It silently replaces a preserved center with a local equivalent.
4. It treats a wrapper or witness line as a primary center without evidence.
5. It flattens a preserved conflict into a single local solution without recording the contradiction.
6. It uses the newest line as default truth when the atlas says another preserved center is stronger for that question class.

Examples:
- inventing a fresh activation engine without consulting Victus/Gemini and the activation-authority judgment
- inventing a new template-development theory without consulting the historical exact-match `TEMPLATE_DEVELOPMENT` center
- treating current branch startup/routing law as if it already equals historical activation-authority execution

---

## Decision rule

When uncertain, classify the work through these questions:

1. Is the burden already preserved somewhere in the estate?
2. Is the proposed work restoring, comparing, or reconciling that burden?
3. Or is it bypassing preserved centers and creating a local substitute?

If the answer to question 3 is yes, the work should be treated as illegal reinvention.

---

## Operational use

This judgment should be applied whenever a future work packet proposes:
- a new protocol
- a new template
- a new role/domain/rank surface
- a new manager/orchestrator/runtime center
- or a significant reinterpretation of center status
