# Stop and Reassess Lifecycle

## Scenario

Question begins as a current-branch bridge repair, but widened evidence suggests the question was misframed.

## Step 1 — Kernel entry
- question class: `current_branch_bridge_repair`
- default center: current branch
- default posture: `bridge_repair`
- default output class: `bridge_repair_note`
- default landing boundary: `current_branch_bridge_repair_only`

## Step 2 — Widening pressure appears
During consultation, the work starts needing:
- historical meta-template comparison
- activation-center comparison
- compactness comparison

## Step 3 — Trigger detection
The following reassessment triggers fire:
- T1 — question-class instability
- T3 — output mismatch
- T4 — landing-boundary mismatch

## Step 4 — Lawful outcome
Do **not** continue as bridge repair.
Do **not** widen indefinitely.

Return to the kernel with:
- stop-and-reassess recommendation
- new likely question class: `recovery_gap_unresolved_contradiction`
- recommended new posture: `recover_first`

## Step 5 — Result
The kernel reclassifies the work instead of letting it continue under a false local framing.
