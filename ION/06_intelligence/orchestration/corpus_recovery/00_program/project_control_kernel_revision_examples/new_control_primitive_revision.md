# New-control-primitive kernel revision example

## Trigger
A repeated class of lawful work cannot be handled honestly because the frozen kernel lacks any explicit rule for revision approval depth.

## Why thaw is allowed
A preserved-center check and repeated reassessment show the frozen kernel is missing a control necessity, not just a wording defect.

## Change class
`new_control_primitive`

## Widening
Limited widening allowed to consult preserved centers and confirm that the missing control primitive is genuinely absent.

## Expected result
Add the smallest new control primitive required, update the manifest, and re-freeze with an explicit judgment.

## Why this example matters
It shows the highest-risk normal revision path and why it requires stronger evidence than simpler change classes.
