# Project Control Kernel Return Conditions

## Purpose

The project control kernel now governs how work starts and whether it may:
- stay inside the kernel,
- widen into the broader atlas or preserved centers,
- or stop and reassess.

This file adds the missing companion rule:
**when and how widened work returns under kernel control.**

Without this, widening can become a new drift channel.

---

## Rule

Widening is lawful only if it remains **kernel-returnable**.

That means widened work should always preserve enough discipline that it can return to the default project control kernel with:
- a declared result type,
- a declared landing boundary,
- a declared conflict posture,
- and a declared next-step recommendation.

If widened work cannot do that, it should stop and reassess rather than continue widening indefinitely.

---

## Canonical return conditions

A widened path may return to the kernel when one or more of these conditions is true:

### R1 — Sufficient consultation return
The widened center has answered the narrow question well enough that the result can now be expressed as:
- a consultation answer,
- comparison judgment,
- recovery note,
- executable receipt,
- conflict entry,
- or lineage update.

### R2 — Boundary-complete return
The widened task has reached the landing boundary that was declared at work start.
There is no lawful need to widen further.

### R3 — Conflict-identified return
The widened work has exposed a conflict that cannot be resolved inside the current pass.
It should return to the kernel as:
- conflict entry,
- missing-evidence escalation,
- or recover-first recommendation.

### R4 — Completion-for-this-pass return
The widened work has achieved its pass-level completion condition, even if the wider organism remains unresolved.
It should return with the explicit statement:
- what is now complete enough,
- what remains unresolved,
- and what would justify the next pass.

### R5 — Misframing-detected return
The widened path reveals that the original question class, posture, or output class was wrong.
It should return to the kernel for reclassification rather than continue under a distorted framing.

### R6 — Reinvention-risk return
The widened work discovers that continuation would likely drift into local invention or silent supersession.
It should return immediately for kernel-level reassessment.

---

## Return artifact minimum

A widened path should return with at least:
- original question class
- widened center consulted
- posture used
- answer/output class reached
- landing boundary reached or blocked
- conflict posture
- return condition satisfied (R1–R6)
- recommended next move:
  - close
  - compare further
  - recover first
  - bridge repair
  - bounded extension
  - stop

---

## Default rule

When in doubt:
**return earlier rather than widen longer.**

The control kernel exists to keep widened work bounded.
It is not a decorative entry surface.
