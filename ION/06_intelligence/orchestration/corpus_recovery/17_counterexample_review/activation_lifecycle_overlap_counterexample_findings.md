# Activation/lifecycle overlap counterexample findings

## Purpose
Compress the main findings from the first adversarial review pass against the activation/lifecycle promotion candidate.

## Main findings

### 1. The strongest remaining confusion is still "entry" language
The word *entry* can tempt a reader to treat activation entry, lifecycle entry, runtime shell entry, and carrier invocation as one event.

That compression is unsafe.
The candidate remains healthiest when the repo preserves a strict sequence:

1. candidate work is prepared upstream
2. activation decides whether enactment may begin
3. lifecycle governs executor behavior after lawful crossing
4. runtime/service shells operationalize the carrier-specific invocation
5. settlement and receipt law record bounded outcome

### 2. Continuation language must stay subordinate to prior lawful crossing
The review set is strongest when continuation is read as bounded restoration, replay, or controlled re-entry under continuity law.
It becomes unstable when continuation is allowed to behave like a free-standing second activation center.

### 3. Denial provenance matters as much as denial itself
Several counterexamples collapse because a refusal is stated, but not attributed clearly enough.
The candidate therefore still needs concrete receipt examples that preserve whether a stop occurred because of:
- failed activation authority
- lifecycle prerequisite failure
- stale/invalidation event
- carrier/runtime entry failure

### 4. Carrier symmetry is still a review requirement, not yet a demonstrated achievement
The current candidate can state carrier invariance, but it has not yet shown it through worked examples.
That keeps thaw readiness out of scope for now.

## Resulting judgment
The promotion candidate is stronger after counterexample naming, but the repo still needs:
- one carrier-crossing worked example set
- one install-path note for possible active-law landing
- one receipt/settlement example set binding activation, lifecycle, continuation, and packet law together
