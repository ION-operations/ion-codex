# Pass 30 — Kernel Governance Cadence Note

## What this pass adds
- a governance cadence for reviewing the frozen default project control kernel
- explicit kernel health checks
- a governance schedule
- a review record template
- a smallest governance card for the kernel itself

## Why it matters
The project already had a frozen default control kernel and a lawful thaw path. What it still needed was a disciplined answer to: when should the kernel actually be reviewed, and how should that review be recorded before anyone even proposes a revision?

This pass adds that missing governance rhythm so the kernel is neither left to drift through neglect nor thawed through convenience.
