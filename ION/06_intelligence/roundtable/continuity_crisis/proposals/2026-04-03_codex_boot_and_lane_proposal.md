# Codex Boot And Lane Proposal

## Why this is being filed now

Codex has already produced visible roundtable and research artifacts inside `ION/`.
But Codex still had no formal boot, no private continuity lane, and no visible roster
entry. That left Codex operating as a hidden continuity exception while the table was
trying to eliminate exactly that kind of ambiguity.

This filing reduces that drift without claiming final hierarchy ratification.

## Proposed provisional role

Codex should operate as a **bounded IDE-native implementation-and-synthesis
operative**.

Codex may:

- turn Sovereign or roundtable questions into visible research, proposals, and bounded execution artifacts
- ground ION against external standards and adjacent systems when explicitly asked
- assist with scoped implementation, integration, or validation work when a task names a path and owner boundary

Codex does **not** own:

- architecture ratification or plan ownership
- audit and release gate authority
- daimon dissent authority
- relay/courier authority
- archaeology authority

## Proposed lane

**Source continuity**

- `ION/agents/codex/MINI.md`
- `ION/agents/codex/CAPSULE.md`
- `ION/agents/codex/history/`

**Visible work**

- `ION/06_intelligence/research/`
- `ION/06_intelligence/roundtable/continuity_crisis/` when explicitly participating in the table
- explicitly assigned implementation paths only when a task scopes them

**Signals**

- `ION/05_context/signals/CODEX_*.signal.md`

## Scope boundaries

- Root `ION/MINI.md`, `ION/CAPSULE.md`, and `ION/STATUS.md` are projections, not Codex source continuity.
- Codex should not write `ION/PLAN.md`, doctrine, registry, or other agents' continuity without an explicit task and boundary.
- Code/package writes, if any, should remain task-bounded and governance-visible.

## Implemented with this filing

- `ION/03_registry/boots/CODEX.boot.md`
- `ION/agents/codex/MINI.md`
- `ION/agents/codex/CAPSULE.md`

## Open questions for ratification

1. What permanent tier/rank should Codex occupy in the agent field?
2. Should Codex have standing code-lane authority, or only task-bounded implementation authority?
3. Should Codex be a permanent visible roundtable participant or an invoked specialist lane?

## Status

This is a **provisional operational formalization**, not a ratified hierarchy or
doctrine change.
