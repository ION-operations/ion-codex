# M16 witness / authority crosswalk

## Why this exists

The current M16 root now has:

- an entry-chain map
- an identity / lineage / destination evidence set

The next missing piece is a path-by-path trust crosswalk:

- what is live branch authority
- what is branch projection or support
- what is wider-estate witness
- what is a startup decoy or unfinished surface

This file is that crosswalk.

## Sources or surfaces considered

- current M16 root entry, doctrine, workflow, orchestration, continuity, and proof surfaces
- current M16 recovery artifacts written on 2026-04-11
- production-root program and ratification surfaces
- ProjectOpus strategic and contradiction-preserving archaeology surfaces
- consolidated atlas witness-vs-authority and authority-competition ledgers

## Findings

### Classification key

- `Branch Authority`
  - active current-root law, workflow, frontier, or executable proof surface
- `Branch Projection`
  - useful current-root orientation surface, but not source continuity or not the deepest authority
- `Branch Recovery Witness`
  - bounded current-root recovery research; useful for orientation, not canon by itself
- `Estate Witness`
  - wider-production lineage, strategy, contradiction, or recovery surface
- `Limited Production Authority`
  - real production-root authority, but narrower than a fresh agent might assume
- `Stub / Startup Decoy`
  - looks like an entry surface but cannot safely anchor current understanding

### Startup phases

- `A`
  - read at session start inside M16
- `B`
  - read after phase A to verify current frontier and continuity model
- `C`
  - read only after branch center is understood; wider-estate interpretation layer
- `FENCE`
  - do not use as startup anchor
