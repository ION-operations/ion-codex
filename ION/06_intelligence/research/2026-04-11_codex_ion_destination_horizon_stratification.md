# ION destination horizon stratification

## Why this exists

The recovery lane now has bounded answers to:

- what ION is
- where ION came from

The next missing answer is where ION is going.

This file gives that answer by separating three horizons that keep getting
flattened together:

- the current-generation finish line
- the immediate post-M16 frontier
- the far-horizon platform vision

## Findings

### 1. Current-generation destination

The strongest bounded current-generation destination is not "full OS."
It is the current M16 generation definition of done.

The acceptance matrix says ION is done at this generation only when it satisfies
all of the following:

- canonical legibility
- executor neutrality
- manual/automatic equivalence
- bounded externality
- multi-agent continuity
- parallel boundedness
- horizon intelligence
- scheduler explicitness
- operational trust
- rehearsed proof
- extension readiness

The roadmap sharpens that into one practical finish-line sentence:

- a capable executor can enter from canonical artifacts
- understand the lawful next step
- carry one bounded step through an approved carrier
- hand off without hidden context
- preserve future structure honestly and traceably

So the correct current-generation destination is:

- a lawful continuity kernel that is demonstrably handoff-capable,
  carrier-neutral, and proof-backed

### 2. Immediate post-M16 frontier

The post-M16 handoff says the current immediate frontier is not abstract.
It is specific:

- M16 landed direct executor-entry rehearsal from the handoff capsule
- M16 did not land executor-start packet materialization
- M16 did not land hidden continuation expansion
- M16 did not land automatic entry dispatch

The next bounded workload is therefore:

- M17 handoff-capsule executor-start packet materialization

The M17 workload plan keeps this tight:

- materialize one executor-start packet from a successful M16 rehearsal
- preserve linkage back to the capsule / activation / continuation chain
- persist one executor-start materialization receipt

and it explicitly forbids:

- new planner behavior
- autonomous dispatch widening
- shadow continuation systems

So the correct immediate destination is:

- continue the current branch one bounded architecture center at a time, with
  M17 as the live next trust gap

### 3. Near platform destination

The strategic direction witness says the immediate larger platform goal is:

- build best ION v3

and defines that as the foundation for everything else.

At this horizon, the project is trying to reach a strong self-hosting ION that:

- uses raw API / local control rather than depending on IDE habitat as a crutch
- can feed context into other builders
- can operate as a real runtime center

This is larger than the current M16 finish line, but smaller than the final OS
dream.

So the near platform destination is:

- a definitive ION v3 / raw-runtime kernel platform that can stand on its own
  and power other tools

### 4. Far-horizon destination

The strategic roadmap preserves the larger long-range sequence:

- best ION v3
- use ION context powers to boost agents
- VS Code fork
- ground-up IDE
- Linux distro
- full OS

The evolution-orchestration north star adds the framing:

- raw-runtime ION/Aether becomes the permanent kernel
- IDEs become clients until ION surpasses them

So the correct far-horizon destination is:

- a full AI-native operating environment in which ION/Aether is the governing
  runtime rather than an add-on inside somebody else's habitat

### 5. Constraint layer

The horizon-separated answer above is necessary because the production-root
authority is still limited.

T32 does not certify:

- daemon autonomy
- full production
- whole-platform completion

It only activates doctrine / kernel / registry for a limited production-root
scope.

The roadmap also explicitly warns against:

- widening into swarm behavior first
- letting product vision outrun proof

So the correct destination answer must preserve the distinction between:

- what the current M16 branch can truthfully claim now
- what the next bounded frontier is
- and what the larger strategic vision still points toward

## Implications

1. The correct answer to where ION is going is horizon-labeled, not singular.

2. For current M16 implementation work, the authoritative destination is:
   current-generation definition of done plus the explicit M17 next workload.

3. The ION v3 / IDE / OS arc is real strategic witness, but it is not the
   immediate implementation authority for this branch.

4. Any future destination answer should specify whether it is speaking about:
   current-generation completion, immediate branch frontier, near platform
   target, or far-horizon organism vision.
