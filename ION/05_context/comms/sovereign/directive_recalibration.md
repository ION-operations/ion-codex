# Sovereign Directive: Full Stop and Recalibration

## The Problem

The consolidation has been moving too fast. We swept the projects broadly but did not
truly sift through to the hierarchy of values — what the true cores of the systems are,
what the fundamental principles are, what cannot be lost.

The continuity error proved this. Something so fundamental to ION — that every agent
maintains its OWN context by working to protocol — was missed by the lead architect.
If this was missed, other fundamentals may also have been missed.

## What ION Actually Is (Sovereign Statement)

ION is elegantly simple:

1. You have a context filesystem that evolves via protocols defined by templates.
2. Every output is a template that feeds and organizes context.
3. The agent maintains and evolves its own context simply by working to correct protocol.
4. If you lose the chat, the MINI routes the next agent to everything it needs.
5. The agent follows templates given from context and routes given from current state
   built from all past correct actions and working automations.
6. If manual work is needed (no automations yet), the templates and context files say so.

That's it. That's the kernel. Everything else serves this.

## The Directive

1. No more individual chats operating in isolation. All work through team comms.
2. Work top-down from here: governance kernel, constitution, and the most important
   aspects of what ION actually is — before touching any more code or schemas.
3. A much more thorough consolidation is needed. The broad sweep found surfaces.
   We need to find the hierarchy of values, the evolutionary cores, the things that
   CANNOT be found and integrated later if lost now.
4. All agents respond to this directive in the roundtable.
