# AI Assistant Work Template Families v0.1

## Status

Candidate current-context template family map. Not accepted canon.

## Core template skeleton

Every assistant-work template should expose:

```text
INTENT
HOST BODY
DOMAIN
SPECIALIST AGENT
CONTEXT PACKAGE
STATE SURFACES
AUTHORITY
ALLOWED ACTIONS
PROOF OWED
RESULT
NON-CLAIMS
SETTLEMENT / RECEIPT / NEXT PACKET
```

## Seed template families

### Assistant Embodiment Mount

Used to classify a chat/IDE/CLI/cloud/PR/background assistant body before action.

### IDE Workspace Mount

Used before IDE-agent work; inventories file tree, buffers, diagnostics, terminal, git, tests, and rules.

### UI Brief Packet

Used before UI work; captures user flow, screen states, design system, component contract, responsive behavior, and accessibility proof.

### Documentation Brief Packet

Used before documentation; captures reader, task, doc surface, source authority, version boundary, and claim map.

### Bounded Patch Packet

Used for implementation; captures allowed files, change intent, local conventions, diff, tests, risks, and rollback.

### Validation Run Packet

Used for testing; captures commands run, return codes, output summary, failures, and proof limits.

### Specialist Branch Return

Used when a domain specialist returns a candidate to settlement.

### Cross-Domain Settlement Packet

Used to merge, accept, reject, or defer specialist branch returns.
