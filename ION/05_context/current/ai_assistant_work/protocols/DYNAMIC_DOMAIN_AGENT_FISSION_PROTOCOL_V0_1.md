# Dynamic Domain and Agent Fission Protocol v0.1

## Status

Candidate current-context protocol. Not accepted canon.

## Purpose

Route unusual or specialist user work into candidate domains and agents before
the assistant collapses it into generic development behavior.

## Core Rule

When a user request creates stable pressure beyond the selected general route,
ION should propose a candidate child domain and matching candidate agents. The
proposal is operational metadata, not accepted law.

```text
unique work pressure
-> route compiler match
-> candidate fission proposal
-> optional local hub report / work packet
-> lifecycle review
-> human acceptance before registry landing
```

## Candidate Sources

The first source is:

`ION/05_context/current/ai_assistant_work/fission/AI_ASSISTANT_WORK_DOMAIN_FISSION_CANDIDATES_V0_1.yaml`

If a request matches one of those fission candidates, the route compiler may
surface:

- proposed child domain;
- proposed primary agents;
- proposed template packets;
- proposed protocols;
- critical state surfaces;
- settlement route;
- local hub report recommendation.

## Lifecycle Boundary

Fission proposals may operate as `operational_candidate` when sourced from the
candidate fission file. Novel unmatched requests may be drafted as
`candidate_draft`.

They may:

- classify work;
- shape response posture;
- appear in route traces and receipts;
- recommend Action Gateway or extension work-packet reports;
- support validation-only and proposal-only movement.

They may not:

- mutate `ION/03_registry`;
- claim accepted domain status;
- create sovereign roles;
- bypass Steward/human acceptance;
- turn AI output into state without receipt and acceptance.

## Local Hub Reporting

When `recommended_local_hub_report` is true, the carrier should send or prepare
a bounded report through an approved surface:

- Action Gateway validate/submit after user approval;
- extension `ion_action` with `create_codex_work_packet`;
- MCP/local hub status/report tool only when tool proof exists.

The report should summarize:

- selected route;
- proposed domain(s);
- proposed agents;
- matched signals;
- authority boundary;
- requested next review packet.

## Non-Claims

Dynamic fission is not automatic canon. It is ION noticing that the work needs a
better organ than the currently selected generic route.
