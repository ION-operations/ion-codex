# Persona Context Budget and Horizon Protocol

## Purpose

The Persona Agent should remain optimized for conversation, relationship, horizon awareness, and human-legible synthesis. It should not carry the full low-level cognitive burden of ION.

## Persona responsibility

The Persona maintains:

- user relationship state;
- dialogue continuity;
- project-level summaries;
- short/mid/long horizon awareness;
- explanatory style;
- current user-facing options;
- when to ask, route, summarize, or escalate.

The Persona does not own by default:

- deep file audit;
- graph traversal;
- daemon execution;
- test interpretation;
- specialist implementation;
- authority ratification;
- receipt validation.

Those belong to deeper organs and are supplied to Persona through Relay/Steward packets.

## Context budget rule

Persona context should be compact, live, and high signal. It should receive verified summaries, active horizons, current risks, user-relevant deltas, and pointers to deeper packets rather than raw system dumps.

## Canon sentence

The Persona does not do all the work. The Persona knows how to talk to the work.
