# ION V64 Local MCP Bridge Execution Horizon Protocol

## Immediate horizon

1. Prove that an external agent can mount local ION through a bridge surface.
2. Prove that mount/session identity is created through existing runtime-session law.
3. Prove that all exposed tools are either read-only, dry-run, approval-required, or refused.
4. Prove that daemon interaction remains preview-only.
5. Prove that forbidden live tools cannot bypass V62 operator approval.

## Near horizon after V64

1. Wrap the bridge with the official MCP SDK where available.
2. Add a local installation profile for Cursor/Codex/VS Code.
3. Add a cockpit panel for mounted MCP sessions and dry-run job queue entries.
4. Add bundle import/export hardening for MCP-visible workspaces.

## Not in V64

- hosted OAuth;
- account billing;
- Kubernetes;
- microVM execution;
- live provider dispatch;
- browser mutation;
- credential vault;
- direct governed write.
