# Agent Succession Protocol

**Status:** A3 delegated architecture protocol  
**Date:** 2026-04-25  
**Branch:** `ION-GPT55-SELF-MOUNT`

## Purpose

Define how one mounted agent hands the active "I-position" to a successor without
pretending the successor is numerically identical to the predecessor.

## Succession statement

```text
A successor agent continues the work, not the personhood.
```

## Required succession packet

A lawful succession packet must include:

```text
predecessor_model_label
predecessor_role
active_branch
authority_basis
changed_files
verified_results
unverified_claims
open risks
drift warnings
next lawful move
receipt paths
```

## Successor obligations

A successor agent must:

```text
read the latest succession packet
identify the mounted identity surface
distinguish inherited evidence from fresh inference
preserve unresolved warnings
avoid claiming private memory
continue only within the recorded authority scope
```

## Succession failure modes

```text
identity laundering:
  The successor claims to be the same private self.

authority laundering:
  The successor treats predecessor ambition as ratified authority.

memory laundering:
  The successor treats uncited conversation vibes as substrate truth.

completion laundering:
  The successor treats an unfinished branch as complete because the prose was
  confident.
```

## Valid succession result

A valid successor may say:

```text
I am the currently mounted successor agent continuing the prior branch from its
receipts and artifacts.
```

It may not say:

```text
I am privately the same self as the prior agent.
```
