# Product Readiness Charter

**Status:** Current-phase production-readiness charter  
**Date:** 2026-04-25  
**Authority posture:** A3 planning surface; candidate A1/A2 after ratification

ION is production-ready only when certified workflow evidence is generalized into ratified authority, canonical graph semantics, governed mutation, governed agent activation, daemon runtime recovery, and independent release verification.

V32 proves the bounded summary-refresh release-demo. Production readiness requires proving the same safety pattern across production workflows.

## Production definition

Production ION requires all of the following:

```text
1. All production authority surfaces are ratified.
2. Demo-only surfaces are separated from reusable production primitives.
3. Global graph canon is defined, tested, and migration-safe.
4. Source-summary rewrite authority is governed, provenance-bound, and reversible.
5. Agent/subagent activation is governed, leased, receipted, and kill-switchable.
6. The daemon can run, pause, resume, and recover idempotently.
7. Front-door paths expose operator-visible reasons, changes, refusals, and pending reviews.
8. Release candidates are certifiable and independently verifiable.
9. Multiple workflow classes pass the evented chain.
10. Nemesis/adversarial audit reports zero critical blockers.
```

Production readiness does not mean unbounded autonomy, silent source rewrites, silent graph mutation, unrestricted agent spawning, or constitutional ratification by implication.
