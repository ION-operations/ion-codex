# Self-Mount Graph Integration Protocol

## Purpose

V38 binds the GPT55 self-mount runtime primitives into ION's context graph
substrate as queryable, bounded projection state.

The graph integration records three executable self-mount families:

1. runtime identity envelopes,
2. self-surface drift assessments,
3. agent succession packets.

## Authority posture

This projection is A3 branch-local graph state. It does not ratify global graph
canon, production migration authority, or production readiness. It exists to
make the self-mount lineage inspectable and recoverable.

## Required graph relations

- a mounted agent identity envelope is constrained by the drift gate;
- a drift gate report evaluates a self-surface claim;
- a succession packet inherits evidence from a predecessor envelope;
- a successor envelope may continue a task from a predecessor packet;
- production-readiness blockers constrain all self-mount graph projections.

## Queryability goal

A future agent should be able to ask: which identity envelopes existed, which
self-claims were blocked or permitted, what packet carried the handoff, what
evidence was inherited, what authority was active, and which production blockers
remained unresolved.
