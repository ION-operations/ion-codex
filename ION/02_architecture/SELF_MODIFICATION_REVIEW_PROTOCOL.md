# SELF-MODIFICATION REVIEW PROTOCOL

**Status:** Current-phase operational proposal  
**Date:** 2026-04-24  
**Authority posture:** A3 until reviewed  
**Purpose:** Define review requirements for any case where ION changes ION.

---

## 1. Scope

Self-modification includes changes to:

```text
doctrine
constitution
protocols
templates
registries
agent role canon
graph substrate law
kernel behavior
approved context rules
runtime proof posture
public product claims
```

---

## 2. Review principle

```text
No self-modifying change may be reviewed only by the same burden that authored it.
```

A builder can propose. A reviewer must review. A verifier must verify.

---

## 3. Depth classes

Uses the D0-D5 classes from `DOCTRINE_EVOLUTION_PROTOCOL.md`.

For D4-D5, at least one contradiction-oriented role must participate:

```text
Vice or Nemesis
```

For D5, donor-line review is mandatory:

```text
Vestige or equivalent historical recovery pass
```

For runtime claims, implementation review is mandatory:

```text
Mason or equivalent code/test pass
```

---

## 4. Required outputs

```text
self_modification_review_record
contradiction_review_record
runtime_mapping_record when applicable
test_guard_record when applicable
verdict_record
receipt
```

---

## 5. Verdicts

```text
ALLOW_BOUNDED
ALLOW_PROVISIONAL
HOLD_FOR_REVIEW
HOLD_FOR_EVIDENCE
ESCALATE
REJECT
```

---

## 6. Forbidden patterns

```text
author self-certifies
beautiful explanation becomes canon
runtime claim lacks test
historical evidence is ignored
registry is updated without receipt
role authority widens by wording
template behavior changes without metadata contract
```
