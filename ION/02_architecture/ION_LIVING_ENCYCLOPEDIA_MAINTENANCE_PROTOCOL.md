# ION Living Encyclopedia Maintenance Protocol

## 1. Purpose

The ION Production Encyclopedia is not a detached explanation document. It is a maintained ION state object whose job is to preserve the current reader spine, historical doctrine, version/gate namespace, proof boundaries, and next lawful implementation path.

The encyclopedia must be updated whenever the accepted runtime stack, branch map, production verdict, context law, template law, agent law, cockpit law, or carrier boundary changes.

## 2. Authority boundary

The encyclopedia may:

```text
explain current state;
preserve historical doctrine;
summarize inspected artifacts;
declare claim boundaries;
record version/gate namespace;
produce handoff prompts and next-lawful-move guidance;
```

The encyclopedia may not:

```text
ratify production readiness;
claim tests that were not run;
treat overlays as full roots;
treat candidate files as live implementation;
treat chat-only correction as artifact correction;
override source files, manifests, receipts, or tests;
```

## 3. Required update triggers

An update is required when:

```text
1. a new runtime base is accepted;
2. an overlay is accepted into the current stack;
3. the canonical mount/run command changes;
4. the agent context package law changes;
5. template/action proof law changes;
6. Steward integration law changes;
7. UI/cockpit proof surfaces change;
8. production-readiness state changes;
9. a candidate implementation becomes live;
10. a prior encyclopedia current-state statement becomes stale.
```

## 4. Required sections

Every future update must include:

```text
CURRENT_STATE_OVERRIDE
SOURCE_ARTIFACTS_AND_HASHES
CLAIM_LEDGER
WHAT_REMAINS_TRUE_FROM_PRIOR_BODY
WHAT_IS_STALE_OR_HISTORICAL
VERSION_AND_GATE_NAMESPACE_MAP
RUNTIME_PROOF_CLAIMS_AND_NON_CLAIMS
NEXT_LAWFUL_MAINTENANCE_MOVE
```

## 5. Integration with context packages

The living encyclopedia must be routeable from all serious agent context systems. It should appear as a route-deeper source for:

```text
role.steward
role.relay
role.scribe
role.context_cartographer
role.runtime_cartographer
role.nemesis
lead_dev_carrier_context_control_surface
```

It should not be loaded in full by default. Instead, context packages should load the current-state supplement and route deeper into the preserved historical body only when lineage, doctrine, or contradiction review requires it.

## 6. Update proof

A valid encyclopedia update must write:

```text
1. updated encyclopedia file;
2. maintenance protocol or protocol amendment if process law changed;
3. manifest with source artifacts and hashes;
4. signal/receipt;
5. FILES_ADDED list for overlay visibility.
```

## 7. Failure modes

The maintenance process must reject:

```text
stale latest-state claims;
silent branch-number collision;
documentation coherence inflated into runtime proof;
focused evidence inflated into full-suite proof;
chat apology without artifact repair;
manual update not reflected in repo paths;
```
