# RUNTIME REPORT CROSSLINK PROTOCOL

## Purpose

Define a bounded, read-only cross-link layer over downstream runtime-report witness surfaces so operators can move more directly among related artifacts, dashboards, ledgers, summaries, and rollups without promoting any of those outputs into kernel truth.

## Governing rule

Cross-link outputs are **navigation witness only**.
They remain downstream from:

1. runtime-report artifacts
2. governance reflections
3. governance aggregation
4. visibility projections
5. navigation packets
6. browser views

They do not become doctrine, route authority, runtime authority, or store/index/graph truth.

## Allowed targets

A lawful F3 cross-link packet may reference only existing downstream generated or witness surfaces, including:

- artifact report paths
- packet-index paths
- operator dashboard paths
- governance ledger paths
- governance summary paths
- system ledger paths
- operator rollup paths

## Read-only law

F3 may:

- resolve related paths
- verify whether those paths exist
- render cross-link packets or browser sections
- emit read-only markdown/json link maps

F3 may not:

- mutate target files as part of traversal
- treat linked witness files as authority
- open a daemon loop or polling surface
- create hidden routing or execution side effects

## Output classes

F3 outputs must be marked as:

- `GENERATED_STATE`
- `READ_ONLY`
- downstream witness/navigation material

## Minimum packet content

A lawful cross-link packet should include:

- packet index path
- optional operator dashboard path
- per-entry crosslink targets
- existence witness for each target
- explicit downstream boundary statement

## Integration rule

Browser rendering may surface F3 crosslinks directly, but doing so does not change the authority class of the browser surface.
