# Context Graph Substrate Protocol

## Core law

ION is a living context graph operated by lawful agents through templates.

## Definitions

The context graph is the governed graph of typed contextual nodes, typed edges, context packages, graph regions, agent jurisdictions, templates, receipts, and settlements through which ION preserves and evolves continuity.

A context node is a durable meaning-bearing unit. A context node may be represented by a file, record, packet, receipt, template, protocol, semantic identity, domain, work unit, signal, memory, decision, test, or external-return witness. A node is not merely a text chunk. It must have identity, type, authority posture, provenance, graph relations, ownership/jurisdiction, and status.

A context edge is a typed relation between nodes. Edges define dependency, provenance, contradiction, authority, implementation, evidence, ownership, routing, supersession, activation, and settlement relations.

A context package is a bounded graph neighborhood assembled for a specific lawful step, role, subagent, handoff, review, or continuation. A context package is not a dump. It is a governed projection of the graph.

A graph region is a coherent subgraph under a domain, role, protocol family, package family, implementation module, historical lineage, or research field.

A graph custodian is an agent or subagent responsible for maintaining awareness, evidence discipline, and lawful mutation within a graph region.

A settlement is a fan-in operation that returns branch/subagent results into parent graph continuity as proposals, decisions, receipts, or updated node/edge relations.

## Fundamental invariant

No agent is expected to hold all of ION. The graph holds continuity. Agents hold bounded graph regions. Manager agents call graph-region specialists and synthesize their settlements.

## Relation to templates

Templates are graph-operation law. They define how nodes are read, assembled, mutated, reviewed, settled, and written back.

## Relation to context windows

ION does not pursue infinite context by loading everything into one prompt. ION achieves practical infinite context through bounded local context plus graph-addressable global continuity plus lawful fan-out/fan-in.
