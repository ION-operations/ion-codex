# TEMPLATE SURFACE EVOLUTION PROTOCOL

**Status:** CURRENT_PHASE / PROVISIONAL_BRIDGE / NOT_FINAL_CANON

## Purpose

Define the lawful path for creating, updating, superseding, restoring, demoting, or retiring template surfaces and template bindings in the active branch.

## Core rule

Template change is itself governed work.
No template surface should be silently mutated by convenience, habit, or lane-local memory.

## Required carrier

Every template-surface mutation should ride a `TEMPLATE_SURFACE_CHANGE` artifact that states:
- what surface is changing
- why the change is needed
- what governing sources justify it
- whether the change is recovered, restated, provisional, or canon-tightening
- what downstream bindings, roles, or context-bank feeds are affected

## Minimum lawful flow

1. Detect a missing, stale, misnamed, or malformed surface.
2. Search live branch and estate history before invention.
3. Record the intended change through `TEMPLATE_SURFACE_CHANGE`.
4. Update the affected template, binding, registry, or semantic/boot surface.
5. Record any role-chassis mount or startup truth correction needed downstream.
6. Emit evidence/witness so a fresh session can understand why the change happened.

## Current-phase note

This protocol is a current-phase governing bridge that restores lawful template evolution pressure while broader constitutional/template canon closure remains open.
