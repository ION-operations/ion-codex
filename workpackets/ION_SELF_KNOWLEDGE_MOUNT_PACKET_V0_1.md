## Objective

Activate ION's existing self-knowledge organs as a routed domain system so future AI carriers stop misunderstanding ION from scattered artifacts.

## Problem

Recent carriers repeatedly misunderstood ION because they entered through scattered repo/package/workpacket surfaces rather than a mounted `ION-about-ION` domain.

Observed failures:

- treating Custom GPT ION as reduced ION instead of full ION adapted to GPT sandbox constraints;
- treating single-carrier sequential mode as the blueprint instead of a host-limited fallback;
- treating full local Codex/API ION as an add-on instead of ION's intended multi-invocation form;
- inferring architecture from current partial broker/queue code instead of routing through ION's self-knowledge roles;
- using language like "ION should have" for systems that already exist in doctrine/context/system cards.
