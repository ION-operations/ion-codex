# Relay Packet — Agent field: hierarchy, skills, realms, ranking, expansion

The Sovereign asked Relay to place the following before the roundtable.

---

## Sovereign intent

The program is doing **a lot** to understand **how ION agents work in operation** (continuity, protocols, roundtable, kernel-shaped behavior). The Sovereign believes the team may also need to consider **very carefully** the **actual full hierarchy**, **skill set**, and **realms** of agents — so that ION has a **complete field**, **ranking**, and **clear intent** per role that can then be **expanded** with **more specialists** and further roles **as needed**.

This is **not** cosmetic org-chart work. Without an **explicit, agreed** map of **who sits where**, **what they may do**, **what realm they own**, and **how seniority / precedence / veto** interact, the system risks **ad hoc** role invention and **authority bleed** even if continuity files are perfect.

The Sovereign notes that **previous ION / API / registry-style builds** already embedded **systems like this** (agent registry, tiers, templates, valid capabilities). That **lineage** should be **mined and reconciled** with the current **boot + lane + roundtable** model — **not** discarded, **not** silently merged without audit.
