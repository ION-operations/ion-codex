# Counterexample review

This layer exists after promotion-candidate review and before any thaw decision.

Purpose:
- test whether a promotion candidate survives explicit adversarial reading
- preserve concrete misreadings and overlap cases that would silently collapse boundary discipline
- force coupled law sets to answer failure cases before they are treated as thaw-ready

Default rule:
a promotion candidate is not thaw-ready merely because its surfaces compose elegantly in positive cases.
It must also survive named counterexamples that stress semantic overlap, stale state, carrier crossing, and continuity disruption.

Current primary set:
- activation authority
- executor lifecycle
- activation/lifecycle interface contract
- runtime session authority
- session queue/dispatch
- API runtime entry
