# Atlas operating layer transition

## Purpose

This note marks the transition from a **recovery program** to an **atlas operating layer**.

Recovery work is still ongoing.
The project is not fully consolidated.
But the atlas is now strong enough to begin acting as a control surface for how future work starts.

## What changed

Earlier passes established:
- archive awareness
- prior-audit awareness
- fractured-core mapping
- values/soul recovery
- executable witness coverage for the strongest missing centers
- a provisional atlas base
- a working atlas base

The current transition adds:
- an explicit future-work gate
- an authority matrix
- a smaller control/read-order layer for future planning work

## What the atlas operating layer is

The atlas operating layer is the smallest set of recovery outputs that future work should consult before it begins.

It does not replace the wider recovery tree.
It governs how the wider recovery tree should be used.

## What it is not

It is not:
- final canon
- final consensus
- permission to erase preserved contradictions
- a claim that reintegration is complete

## Current practical meaning

From this point forward, a future worker should not begin from whichever surface is most convenient.
A future worker should begin from the atlas operating layer, classify the question, select the right preserved center, and only then proceed.
