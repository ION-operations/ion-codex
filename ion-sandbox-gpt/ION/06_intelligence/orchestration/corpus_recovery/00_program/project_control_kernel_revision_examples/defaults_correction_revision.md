# Defaults-correction kernel revision example

## Trigger
A default answer/output mapping repeatedly causes lawful recover-first work to land too narrowly as advisory-only instead of atlas evidence.

## Why thaw is allowed
The defaults are producing repeatable lawful dead-ends. The kernel itself needs a defaults correction.

## Change class
`defaults_correction`

## Widening
Only enough to compare against preserved center evidence and current conflicts.

## Expected result
Update the defaults matrix, confirm landing boundary correction, and re-freeze.

## Why this example matters
It shows how the kernel can be revised without inventing new control primitives.
