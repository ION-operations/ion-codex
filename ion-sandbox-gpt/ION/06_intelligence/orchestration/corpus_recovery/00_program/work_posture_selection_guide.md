# Work Posture Selection Guide

## Purpose

This guide helps future work choose a lawful posture after passing through the atlas gate.

The project historically failed by treating every question like a license to build.
This guide exists to stop that.

---

## Allowed postures

### 1. Consult only
Use when:
- the atlas already answers the question well enough
- the work only needs orientation, not new branch mutation
- the main task is to avoid starting from the wrong center

Typical outputs:
- note
- summary
- no branch mutation beyond small reference updates

### 2. Compare centers
Use when:
- more than one preserved center is relevant
- the key task is discrimination, not mutation
- compactness vs explicitness or historical vs current tension is central

Typical outputs:
- comparison note
- center judgment
- conflict expansion

### 3. Recover first
Use when:
- evidence is still too weak to justify mutation
- a historical center is known but not yet mapped strongly enough
- the project risks false novelty if it moves ahead now

Typical outputs:
- recovery receipt
- profile addition
- executable witness
- missing-evidence reduction

### 4. Bridge repair
Use when:
- the current branch clearly lacks something stronger historical lines preserved
- the task is to repair or reconnect current branch law to a stronger preserved center
- the work is still bounded and explicit about what is being bridged from

Typical outputs:
- bridge note
- repair doc
- narrowly scoped branch update

### 5. Bounded extension
Use when:
- the atlas search and conflict layer support the conclusion that the exact work is genuinely absent
- the extension is small, explicit, and does not silently replace a preserved center

Typical outputs:
- bounded new surface
- explicit non-reinvention record
- landing note

---

## Selection rules

Pick the **least expansive** posture that still answers the question.

Default order of preference:
1. consult only
2. compare centers
3. recover first
4. bridge repair
5. bounded extension

If two postures seem possible, prefer the one that widens the organism less.

---

## Historical anti-patterns this guide is meant to stop

- treating every question as an implementation task
- skipping the compact/historical center because the current branch is easier to edit
- inventing a new local replacement before checking preserved lines
- flattening a conflict into a convenience answer
- widening branch scope when consult-only or compare-only would have been enough

## Relation to canonical question classes

Posture selection should happen **after** question classification, not before.
Use:
- `canonical_future_work_question_classes.md`
- `canonical_future_work_question_class_defaults.csv`

to identify the default starting posture for the class of question being asked.


## Output discipline

Answer/output class comes after posture selection.
Once posture is chosen, future work should select the narrowest answer/output class that still fits the task.
For many question classes, the correct outcome is not a new packet or repair note but a consultation answer, comparison judgment, conflict entry, or recovery note.


## Additional rule

After posture selection, future work should also choose:
- the narrowest lawful answer/output class
- the narrowest lawful landing-boundary class
- the current process position and near/mid/far horizon if the task is non-trivial
