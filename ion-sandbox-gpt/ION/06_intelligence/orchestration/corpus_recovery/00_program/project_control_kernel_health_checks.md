# Project Control Kernel Health Checks

## Purpose

These health checks provide the smallest practical audit of whether the frozen default project control kernel is still functioning as intended.

## Core checks

### H1 — Question routing health
Can a future worker classify the question without obvious ambiguity or local reframing drift?

### H2 — Center selection health
Does the defaults layer still route major question classes toward the strongest preserved center first?

### H3 — Posture health
Are work postures still biased toward the least widening lawful option?

### H4 — Output health
Are answer/output classes still preventing oversized or mislanded results?

### H5 — Landing health
Do landing boundaries still prevent silent mutation of law, atlas, or code surfaces?

### H6 — Widen/return health
Do widened tasks reliably return under declared conditions instead of wandering indefinitely?

### H7 — Reassessment health
Do reassessment triggers fire early enough when work is misframed?

### H8 — Anti-reinvention health
Does the kernel still make illegal reinvention harder than lawful consultation and recovery?

## Health result classes

- healthy
- strained but acceptable
- repeated insufficiency
- revision warranted
- freeze rollback should be considered

## Minimum evidence for a health concern

A health concern should be treated as real when it is repeated, patterned, and attributable to the kernel rather than to simple operator error.
Single confusing episodes are not enough to justify thaw.
