# Proposal-space surface design

This directory holds **proposal-space** surface-design packets derived from the controlled-reintegration deltas.

These surfaces are **not active law**.
They exist to define draft current-line protocol shapes before any thaw, revision, or architecture ratification work begins.

## Boundary

Proposal-space packets may:
- define candidate protocol splits,
- define ownership boundaries,
- define state/object vocabularies,
- and define ratification prerequisites.

Proposal-space packets may **not**:
- install active protocol law,
- claim architectural ratification,
- or mutate live kernel behavior by documentation pressure alone.
