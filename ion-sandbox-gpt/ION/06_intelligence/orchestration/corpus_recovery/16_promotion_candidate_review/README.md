# Promotion-candidate review

This layer exists after quarantined active-law review and before any thaw or active `02_architecture/` installation.

Purpose:
- evaluate whether a quarantined review set is mature enough to become a thaw candidate
- preserve joint promotion criteria for coupled law surfaces
- prevent isolated promotion of one surface that only makes sense with its paired boundary contracts

Current primary set:
- activation authority
- executor lifecycle
- activation/lifecycle interface contract
- runtime session authority
- session queue and dispatch
- API runtime entry

Default rule:
no promotion candidate is treated as thaw-ready until semantic boundaries, failure modes, continuity behavior, and carrier symmetry are jointly reviewed.
