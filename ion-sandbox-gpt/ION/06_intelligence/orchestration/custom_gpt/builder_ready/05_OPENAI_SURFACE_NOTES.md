# OPENAI CUSTOM GPT SURFACE NOTES

These notes reflect the current intended product posture for this build.

## Operational assumptions

- Instructions are finite and should hold compact operating law only.
- Knowledge should hold stable doctrine, not session-growing mutable state.
- Data Analysis should be enabled so the GPT can inspect uploaded bundle files and generate downloadable updated artifacts.
- File-only mode is preferred for the first release posture.
- Actions are deferred until a real external continuity service is justified.

## Practical shell rule

Treat the GPT as:
- shell + doctrine + file operator

Do not treat the GPT as:
- encrypted vault
- ever-growing memory database
- replacement for explicit bundle continuity
