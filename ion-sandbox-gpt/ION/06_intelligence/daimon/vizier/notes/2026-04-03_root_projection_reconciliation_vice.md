# Root Projection Reconciliation

## Thesis

The root continuity story now has the following form:

- the corrected law says the root trio are projections
- the root trio still present themselves as if they are source routing and shared continuity

That is no longer just stale text.
It is an onboarding hazard.

Fresh sessions can still be taught the old model before they ever encounter the corrected one.

## What needs to be true

If root files remain visible and high-traffic during recovery, they must teach the **new**
contract immediately, even before the entire runtime is rebuilt.

The root should not disappear yet.
The root should stop mis-teaching.

## Per-surface reconciliation

### 1. `ION/MINI.md`

### What it currently does

- presents itself as the active routing state for next work
- says Phase 1 is cleared and there is no blocker
- routes readers to root `MINI.md`, `CAPSULE.md`, and `STATUS.md` first

### Why this is dangerous

This is the highest-risk file because it is written in imperative routing form.
It tells a fresh session what reality is.

Right now it still teaches:

- root trio first
- no blocker
- Phase 1 ready

That directly conflicts with:

- continuity roundtable `ACTIVE`
- Nemesis continuity audit `FAIL`
- corrected continuity law
- Vice blockers still outstanding

### Recommended correction

Turn `ION/MINI.md` into an explicitly **Vizier-curated projection** with language like:

- continuity law clarified; operational landing incomplete
- continuity roundtable still active
- root surfaces are projections, not source continuity
- role-specific source continuity lives in each role's source root

And change route order so it points readers first to:

- corrected continuity law
- roundtable synthesis / active reconciliation artifacts
- the reader's own source continuity or role lane, where applicable

### 2. `ION/STATUS.md`

### What it currently does

- acts like a shared agent-sectioned state board
- tells each agent to update its own section
- still contains stale pre-roundtable status narratives

### Why this is dangerous

The corrected law explicitly says:

- `ION/STATUS.md` is a temporary coordination view
- updating shared per-agent sections is the wrong continuity model
- `ION/STATUS.md` is deprecated in favor of source continuity + compiled projection

So `ION/STATUS.md` is currently a live contradiction in both semantics and instructions.

### Recommended correction

Choose one of these and say it plainly:

1. **Temporary projection mode**
   Keep `ION/STATUS.md`, but mark it as Vizier-curated coordination projection and stop telling every role to update its own section.

2. **Deprecation mode**
   Mark it deprecated and route readers to role-private continuity plus signals and intelligence artifacts instead.

Either is acceptable temporarily.
The current mixed story is not.

### 3. `ION/CAPSULE.md`

### What it currently does

- functions as a unified shared work ledger
- still reads like a common continuity spine for everyone

### Why this is dangerous

The file is useful.
The danger is not its existence.
The danger is that it can still be mistaken for each agent's work continuity.

### Recommended correction

Retain it, but relabel it as:

- Vizier-curated system activity ledger
- projection / witness surface
- not any role's private CAPSULE

This preserves visibility without preserving the wrong memory contract.

### 4. `ION/PLAN.md`

### What it currently does

- still says `status: DRAFT`
- still explains IDE/manual mode in language that assumes `MINI.md`, `CAPSULE.md`, and `STATUS.md` are the active continuity surfaces

### Why this is dangerous

`ION/PLAN.md` is not itself continuity, but it teaches the operational worldview.
If the worldview remains old-model while the architecture law has changed, the plan becomes
an authority-looking stale teacher.

### Recommended correction

Add or revise one visible section so the plan says:

- source continuity is private or lane-native by role class
- root files are operator projections during recovery
- `STATUS.md` is temporary / deprecated
- no clone scaling or automation promotion until operational landing is reconciled
