# RUNTIME REPORT GOVERNANCE AGGREGATION PROTOCOL

## Purpose

Define the lawful second-order promotion path from E1 governance reflections into broader operator and system witness surfaces.

## Core rule

Aggregation is permitted only over already-emitted runtime-report trigger receipts and their E1 witness reflections. Aggregation never upgrades those receipts into doctrine, kernel truth, route authority, or runtime authority.

## Allowed surfaces

- `ION/05_context/history/system_ledger.json`
- `ION/05_context/runtime_reports/governance/rollups/`

## Activation conditions

Aggregation remains off by default and requires explicit caller opt-in through trigger policy.

## Boundary

Aggregated rows and rollups are second-order witness material. They improve cross-event operator visibility but remain subordinate to:

1. underlying runtime artifacts
2. trigger receipts
3. kernel store / index / graph truth
4. constitutional and protocol law
