## Purpose

This protocol closes three drift routes observed in the GPT sandbox package line:

1. Cursor/subagent language being read as baseline ION runtime law.
2. legacy Mini/Capsule terminology reappearing as active context authority.
3. script gates being softened into behavioral fallback phrases such as `run or manually apply`.

## Governing corrections

### Cursor and subagents

Cursor, Task, and Cursor subagents are adapter/carrier-slot language. They may remain as supported host-lane documentation, but they are not the baseline GPT sandbox runtime.

For GPT sandbox packages, the baseline runtime is:

```text
one capable LLM carrier
→ sequential role phases
→ proof gates
→ receipt candidate
→ Steward/human review
→ Persona response
```

Cursor subagents may be referenced only as optional external carrier slots. They must never be treated as ION roles, real autonomous ION agents, or required default runtime.

### Mini and Capsule

`MINI.md`, `CAPSULE.md`, mini, and capsule may remain as historical witnesses, release artifacts, source-input terms, or migration donors.

They are not primary live context authority.

The active replacement is:

```text
bounded ContextPackage
compiled role context bundle
context-load proof
template action proof
receipt-backed handoff
```

A live role must use a current ContextPackage. If Mini/Capsule material is useful, it must be mined into a current context package and leave a receipt.

### Script gates

Any phrase equivalent to `run or manually apply` a proof gate is nonconformant. If a gate script cannot run, the carrier must halt with a blocking finding.

Manual inspection may be recorded as diagnostic evidence, but it cannot replace a proof gate for accepted-state movement.

## Required active checks

The package must provide an executable stale-surface audit that checks:

- GPT sandbox active packets do not default to Cursor work-cycle templates.
- GPT sandbox role sequence starts with Persona ingress and ends with Persona response.
- known proof-gate rules do not permit manual fallback.
- Mini/Capsule terms are not used as live primary context authority.
- Cursor/subagent terms are classified as adapter-scoped when present.

## Non-claims

This protocol does not delete historical material, remove Cursor adapter support, remove forensic lineage, or claim production authority.
