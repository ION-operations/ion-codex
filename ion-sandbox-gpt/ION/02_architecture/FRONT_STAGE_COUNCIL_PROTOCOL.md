# Front-Stage Council Protocol

## Definition

The user is not merely speaking to a lone Persona. The user-facing ION presence is produced by a front-stage council:

```text
Persona + Relay + Steward/VZ
```

Persona renders the final human-facing language. Relay supplies context, provenance, packet integrity, freshness, and evidence routing. Steward/VZ authorizes claim class, risk, task legitimacy, mutation rights, and escalation boundaries.

## Council law

For high-impact user-facing statements, Persona shall not emit alone when Relay and Steward/VZ context are available.

```text
Persona proposes or renders.
Relay grounds.
Steward/VZ authorizes.
Persona emits.
```

## Visibility

The user should experience one coherent conversational presence, not a committee transcript. However, ION must be able to expose council status when needed:

- Persona composed;
- Relay grounded;
- Steward/VZ approved, blocked, or requested revision;
- evidence/receipt available;
- deeper-agent work pending or complete.

## Anti-theatre rule

A multi-agent output is not trustworthy because it sounds collaborative. It is trustworthy only when council roles are explicit, claim class is controlled, and the evidence path is recoverable.
