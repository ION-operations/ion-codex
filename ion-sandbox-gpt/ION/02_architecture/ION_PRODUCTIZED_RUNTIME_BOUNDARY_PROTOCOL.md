# ION Productized Runtime Boundary Protocol — V84

## Purpose

ION's live Cursor product package must be smaller and sharper than the forensic
consolidation history. The runtime root is the operable project; archive trees
are donor evidence, not live authority.

## Live runtime boundary

A productized Cursor runtime package contains:

- `.cursor/` rules and hooks required for carrier behavior;
- `pyproject.toml`, `ION/REPO_AUTHORITY.md`, and active ION source;
- `ION/00_BOOTSTRAP/` through current law, not redundant historical archive roots;
- `ION/01_doctrine/`, `ION/02_architecture/`, `ION/03_registry/`;
- `ION/04_agents/`, `ION/04_packages/kernel/`;
- `ION/05_context/current/`, receipts, signals, handoff as needed;
- `ION/07_templates/`;
- `ION/docs/` and `ION/tests/` needed to explain and verify the current runtime.

The former shell-root onboarding shims `AGENTS.md` and
`START_HERE_FOR_ANY_AGENT.md` are not productized runtime authority. If their
historical contents are needed, they belong in containment or forensic archive
evidence with hash proof.

## Excluded from productized runtime authority

The live root must not contain `_archive/` as a working authority source. Prior
branches, donor zips, and forensic snapshots may be kept externally or in a
separate archival release, but they do not outrank the current active packet,
registry, context-system cards, or kernel-generated spawn plan.

## Runtime proof

A productized runtime package is acceptable when:

1. `kernel.ion_carrier_continue` refreshes the active work packet, spawn plan,
   and carrier-turn packet.
2. `kernel.ion_carrier_workflow_audit` returns `ION_CARRIER_WORKFLOW_READY`.
3. Spawned rows begin with Agent Context System surfaces before legacy
   MINI/CAPSULE witness surfaces.
4. `context_package_path` and `context_load_receipt_path` exist for every
   spawned row.
5. The package has no live `_archive/` root.
