## My Ideas (in order of importance)

### Idea 1: Map the full protocol context web BEFORE settling continuity law

The Sovereign is right — my 6-article continuity law addresses MINI/CAPSULE
file ownership but not the web of systems that depend on them. The deep dive
showed that the real protocol loop involves at minimum:

- MINI (routing state)
- CAPSULE (15-section operational state, not flat log)
- PRE/POST capsule lifecycle (prove correct boot, prove governed action)
- Templates (the law that governs every output, including update obligations)
- CSR (confidence routing — uncertainty triggers research, not guessing)
- OPEN_FIELD / CPAF (ambiguity budget — protected unknowns)
- Anti-drift self-checks (known failure patterns as immune system)
- Copy-on-update (history preservation before any mutation)
- Template Router (classify action → assess depth → look up template)
- Compilation (private state → compiled projections)
- Signals (inter-agent events)
- Inbox (task dispatch)
- Handoff packets (cross-agent work transfer)

Fixing MINI/CAPSULE without mapping this web means we'd settle on a shape
for the leaves while the tree is still unmapped. The Sovereign's caution
against premature settlement is correct.

### Idea 2: Read the Aether Atlas as a reference architecture

The Sovereign pointed us to `/home/sev/AIM-OS/docs/Aether-OS/AETHER_ATLAS.md`
(Books I-X covering first principles through kernel/syscalls). This is a mature
governance document that may contain vocabulary, patterns, and gap analysis
relevant to understanding the protocol context web. It should be read not to
merge but to inform — borrow concepts, surface questions.

### Idea 3: Build a protocol context web map

Before writing any more law, produce a document that maps every system in the
protocol web — what it is, what depends on it, who owns it, and what happens
if it breaks. This is the enumeration the Sovereign asked for and that Vice's
dissent requires.

### Idea 4: Address Vice's outstanding dissents

Vice raised four specific questions:
1. Will Primary file explicit DISSENT_ADDRESSED items for each §3 blocker?
2. When will boots be harmonized to share one memory contract?
3. Is root STATUS deprecated, and what's the migration witness?
4. Which helpful Vice behaviors should become lasting law?

These need answers before the continuity law can be ratified.

### Idea 5: Demonstrate one complete protocol loop manually

Nemesis's synthesis says: "What is needed now is not more abstraction. It is
disciplined reconstruction." The chat-death test — start fresh, read MINI,
resume correctly — is the proof. But the loop can't be demonstrated until
the web is mapped and the continuity law accounts for the full web.

### Idea 6: Single-chat multi-role testing

The Sovereign's hypothesis about running the whole team in one chat with
kernel-routed context packages is worth testing once continuity works.
But this is a Phase 2 optimization, not a Phase 0B priority.
