# Full Product Copy Policy

The productized copy must preserve real ION functionality and doctrine. It must not reduce ION to a toy scaffold.

Main runtime copy includes code, tests, doctrine, architecture, templates, registries, UI, integrations, and current state surfaces.

Forensic companion includes bulky historical/runtime evidence that is meaningful but not needed in the operational root by default.

Exclusions are limited to local Git internals, caches, generated build metadata, unsafe local credentials, and similar non-product material.
